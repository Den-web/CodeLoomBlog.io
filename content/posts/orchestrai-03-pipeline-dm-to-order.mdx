## Why “chat” loses

DM input is unstructured. You must compress it into **slots**: product, qty, delivery, payment. One mega-prompt breaks on edge cases and webhook retries.

---

## Theory: stages

```
ingest → extract → normalize → validate → plan → act
```

- **Idempotent** turns (Instagram webhook retry)
- **Fallback**: heuristics, smaller model, human
- AI does **not** write `Order` directly — only through normalized state + gates (part 2)

---

## How we wire it (conversation-first)

- Instagram DM → `Message` → **turn pipeline**
- Chain: extract → `ConversationOrderState` → validators → autofill tools → progress snapshot → **draft gate** → `replyPlan`
- **Invariant:** no `message → Order` jump without a gate

We do **not**:

- One god-prompt for extract + reply + catalog match
- DB writes from browser AI (part 1)
- Ignore duplicate webhooks
