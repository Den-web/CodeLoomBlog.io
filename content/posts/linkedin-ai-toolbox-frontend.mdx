## Categories (how I grouped the tools)

### Design → code

- **v0** — layout and component drafts from prompts
- **Figma + MCP / plugins** — when design system tokens already exist

### IDE / agent

- **Cursor** — repo-wide refactors, tests, docs
- **Copilot / Cody** — inline completion where Cursor is not allowed by policy

### Local / privacy

- **WebLLM** — in-browser models for demos without sending PII to a cloud endpoint

### Quality

- **Playwright + AI helpers** — generate selectors cautiously, always review
- **ESLint / TS strict** — AI output is not exempt from CI

## Rules I follow publicly

1. **Never** paste client secrets or production DB rows into cloud chats
2. Treat model output as **draft** — same as a junior PR
3. Separate **UI hints** from **server decisions** (see [OrchestrAI series](/posts/orchestrai-01-ui-ai-vs-server-ai))
