## Why this post worked

Developers do not need another hot take — they need a **curated pulse** on what actually shipped: React 19, Next.js 15, Node 22, and the tooling around them. The original thread was in Ukrainian for my local network; this page is the English reference on Code Loom.

---

## React 19 — what to care about

- **Actions & form status** — less boilerplate for pending/error states
- **Document metadata** — simpler head management in app router projects
- **Ref as prop** — fewer `forwardRef` wrappers in component libraries

**Practical takeaway:** upgrade paths are smoother when you already use the App Router and strict TypeScript — plan a staging pass for third-party UI kits.

---

## Next.js 15 — ecosystem shift

- **Stable-ish App Router** assumptions for new greenfield work
- **Caching defaults** changed how many teams think about `fetch` — read the migration notes before blaming “random” revalidation
- **React 19 alignment** — check peer dependency matrices on `npm install`

---

## Node 22 — runtime

- LTS cadence matters for **Docker base images** and CI
- Native test runner and permission flags — worth a spike if you still run Jest-only everywhere
